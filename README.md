# Инверсия биометрических эмбеддингов: оценка угрозы приватности

## Описание проекта

Face recognition стал де-факто стандартом для разблокировки мобильных устройств, 
банковской аутентификации и систем контроля доступа. Ключевой компонент таких систем — 
функция эмбеддинга: нейросетевое преобразование, которое сжимает изображение лица 
в числовой вектор размерности 512–1024. Именно на основе этого вектора принимается 
решение об идентичности человека.

Разработчики подобных систем нередко называют хранение эмбеддингов «приватно безопасным»: 
раз исходное изображение не сохраняется, то и биометрические данные якобы защищены. 
Однако этот тезис не подкреплён строгим математическим анализом обратимости функций 
эмбеддинга.

## Проблема

Задача инверсии эмбеддинга (model inversion attack) — восстановление изображения, 
близкого по некоторой метрике к оригиналу, на основе известного вектора — изучается 
с середины 2010-х годов. Mahendran и Vedaldi показали принципиальную возможность 
визуальной реконструкции признаков нейронных сетей, а Dosovitskiy и Brox предложили 
эффективные обучаемые декодеры для этой задачи.

Тем не менее для современных биометрических энкодеров, таких как ArcFace и CosFace, 
оптимизированных по угловым функциям потерь, вопрос о допустимости хранения эмбеддингов 
с точки зрения приватности остаётся открытым.

## Цель работы

Разработать и экспериментально проверить гибридные методы инверсии ArcFace-эмбеддингов, 
ввести корректные метрики оценки угрозы приватности и определить, является ли хранение 
биометрических эмбеддингов безопасной практикой с точки зрения защиты персональных данных.

**Объект исследования:** функции эмбеддинга нейросетевых биометрических систем.

**Предмет исследования:** методы инверсии биометрических эмбеддингов и численные 
характеристики угрозы приватности, возникающей при их хранении.
